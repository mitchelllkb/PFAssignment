2. Initialisation

To initialized a variable is basically to assign values to the variable declared

Declaration by let is possible with or without initializing the variable.
let age; //The variable is declared but not initialized
let age = 23; //The variable is declared and initialized
It's syntax incorrect to declare const as a variable without initializing it.
const hotelName; //It will result to SyntaxError: Missing initializer in const declaration.
The correct way to declare a const is as follow:
const hotelName = "Raffles Hotel";

Variable Initialization by Function

Function returns a value
Variable gets value from a function call.
let total = calculateSum();
Function expression
Variable holds a function definition.
const greet = function(name) { return "Hi " + name; };
Arrow function
Concise function assignment.
const double = x => x * 2;

Variable Initialization by Object

Object literal
Variable holds key-value pairs.
const user = { name: "Alex", age: 30 };
Object from function
Function returns an object.
let config = getSettings();
Object constructor
Using new to create an object.
let date = new Date();

4. Hoisting

Hoisting is a JavaScript behaviour where declarations are moved to the top of their scope (Global or Function) during the compilation phased, before execution.

Hoisting occurs while compiling. If a variable is declared by var and due to its "loose" nature, the value will be read and sent through the function.
var age = 23; // The variable age is declared and assigned with a value of 23.
function getAge()
{console.log(age)};
getAge(); //It will return an output as 23
If a variable is declared by let or const, it will lock itself in Temporal Dead Zone (TDZ) which prevents the function from reading its value.
let age = 23; // The variable age is declared and assigned with a value of 23.
function getAge()
{console.log(age)};
getAge(); //The return output will indicate as a ReferenceError.
